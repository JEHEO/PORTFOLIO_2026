# /review — 코드 리뷰 커맨드

> 이 커맨드는 변경된 코드를 대상으로 프론트엔드 포트폴리오 기준 셀프 리뷰를 수행합니다.
> CLAUDE.md 워크플로우 2단계에서 자동 실행, 또는 `/review`로 수동 실행.

---

## 리뷰 체크리스트

### ✅ 1. 컨벤션 준수

- [ ] 파일명이 규칙에 맞는가?
  - 컴포넌트: `PascalCase.tsx`
  - 훅: `use` 접두사 + `camelCase.ts`
  - 유틸: `camelCase.ts`
  - 라우트 세그먼트: `kebab-case/`
- [ ] 컴포넌트 파일 내 Props 인터페이스가 상단에 선언되어 있는가?
- [ ] `any` 타입을 사용하지 않았는가?
- [ ] 인라인 `style` 속성을 사용하지 않았는가?
- [ ] CSS 모듈 (`.module.css`)을 사용하지 않았는가?
- [ ] `console.log`가 프로덕션 코드에 남아 있지 않은가?
- [ ] 하드코딩된 URL / 매직 스트링이 없는가? (`lib/constants/` 사용)

### ✅ 2. 컴포넌트 구조

- [ ] Server Component가 가능한 경우 `"use client"` 없이 작성했는가?
- [ ] Client Component는 컴포넌트 트리의 말단(leaf)에 위치하는가?
- [ ] `async` 컴포넌트에 `"use client"`를 붙이지 않았는가?
- [ ] 재사용 가능한 컴포넌트는 `components/ui/` 또는 `components/sections/`에 분리했는가?

### ✅ 3. 접근성 (a11y)

- [ ] 이미지에 `alt` 속성이 있는가? (장식 이미지는 `alt=""`)
- [ ] 버튼에 텍스트 또는 `aria-label`이 있는가?
- [ ] 링크에 의미 있는 텍스트가 있는가? (`클릭` 같은 모호한 텍스트 금지)
- [ ] 제목 태그(`h1`~`h6`)의 계층 구조가 논리적인가?
- [ ] 폼 요소에 `label`이 연결되어 있는가?
- [ ] 키보드 포커스가 자연스럽게 이동하는가?

### ✅ 4. 반응형

- [ ] 모바일 퍼스트로 작성되었는가? (`sm:` → `md:` → `lg:` 순서)
- [ ] 주요 뷰포트(모바일/태블릿/데스크톱)에서 레이아웃이 깨지지 않는가?
- [ ] 텍스트가 작은 화면에서도 읽기 편한 크기인가?

### ✅ 5. 성능

- [ ] `<img>` 태그 대신 `next/image`의 `<Image>`를 사용했는가?
- [ ] `<a>` 태그 대신 `next/link`의 `<Link>`를 사용했는가?
- [ ] 무거운 라이브러리를 클라이언트 번들에 포함시키지 않았는가?
- [ ] 이미지에 `width`, `height` 또는 `fill` 속성이 지정되어 있는가?
- [ ] 필요 없는 `useEffect` / 불필요한 리렌더링이 없는가?

### ✅ 6. 코드 품질

- [ ] 중복 코드가 없는가? (3회 이상 반복 시 유틸/컴포넌트로 추출)
- [ ] 함수/컴포넌트가 단일 책임을 가지는가?
- [ ] 주석이 과도하지 않으며, 필요한 곳에만 달려 있는가?
- [ ] TypeScript 타입이 명확하게 정의되어 있는가?

## 리뷰 결과 처리

- 🔴 **차단(Blocker)**: 즉시 수정 후 재리뷰. 다음 단계 진행 불가.
  - TypeScript 오류, 접근성 심각 문제, `any` 타입, 인라인 style
- 🟡 **권고(Warning)**: 가능하면 수정. 사유가 있으면 주석으로 명시.
  - 컴포넌트 분리 미흡, 약한 타입 정의
- 🟢 **정보(Info)**: 향후 개선 고려. 즉시 수정 불필요.
  - 리팩토링 기회, 성능 최적화 제안
